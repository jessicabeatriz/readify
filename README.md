## <span aria-hidden="true">🇧🇷</span> Português

# <span aria-hidden="true">📚</span> Readify 

> Um sistema para gerenciamento de leituras e organização de bibliotecas pessoais, desenvolvido de ponta a ponta com o objetivo de aplicar e consolidar boas práticas de desenvolvimento.

<span aria-hidden="true">🚧</span> **Status do Projeto:** Em desenvolvimento (Fase de Planejamento e MVP).

### <span aria-hidden="true">🎯</span> O Projeto
O Readify é uma aplicação Full-Stack (Monorepo) onde leitores podem cadastrar livros, gerenciar status de leitura (Quero ler, Lendo, Lido, Abandonado) e organizar suas coleções. Mais do que as funcionalidades, o foco deste projeto está na qualidade do código e na arquitetura da solução.

### <span aria-hidden="true">🧠</span> Conceitos e Boas Práticas Aplicadas
Durante o desenvolvimento, os seguintes princípios norteiam as decisões técnicas:
* **Design e Arquitetura:** Princípios SOLID, Clean Code e Programação Orientada a Objetos (POO).
* **Padrões de Projeto:** Arquitetura em camadas (Controller-Service-Repository baseada no MVC) e design de API RESTful usando DTOs.
* **Qualidade e Segurança:** Escrita de Testes automatizados, tratamento global de exceções e autenticação com JWT.
* **Experiência Inclusiva:** Desenvolvimento com foco em a11y, seguindo as diretrizes WCAG para garantir uma experiência acessível e inclusiva.

### <span aria-hidden="true">🛠️</span> Tecnologias Principais
**Backend:**
* Java 25 & Spring Boot 4
* Spring Data JPA & Hibernate
* Spring Security & JWT
* PostgreSQL & Flyway (Migrations)

**Frontend:**
* React + TypeScript
* Vite
* Styled-components
* Figma

## <span aria-hidden="true">🇺🇸</span> English

# <span aria-hidden="true">📚</span> Readify 

> A system for managing readings and organizing personal libraries, developed from end-to-end to apply and consolidate software development best practices.

<span aria-hidden="true">🚧</span> **Project Status:** In development (Planning and MVP phase).

### <span aria-hidden="true">🎯</span> The Project
Readify is a Full-Stack (Monorepo) application where readers can register books, manage reading statuses (Want to read, Reading, Read, Abandoned), and organize their collections. More than just features, the focus of this project is on code quality and solution architecture.

### <span aria-hidden="true">🧠</span> Applied Concepts and Best Practices
During development, the following principles guide technical decisions:
* **Design and Architecture:** SOLID principles, Clean Code, and Object-Oriented Programming (OOP).
* **Design Patterns:** Layered architecture (Controller-Service-Repository based on MVC) and RESTful API design using DTOs.
* **Quality and Security:** Automated tests writing, global exception handling, and JWT authentication.
* **Inclusive Experience:** Development focused on a11y, following WCAG guidelines to ensure an accessible and inclusive experience.

### <span aria-hidden="true">🛠️</span> Main Technologies
**Backend:**
* Java 25 & Spring Boot 4
* Spring Data JPA & Hibernate
* Spring Security & JWT
* PostgreSQL & Flyway (Migrations)

**Frontend:**
* React + TypeScript
* Vite
* Styled-components
* Figma
